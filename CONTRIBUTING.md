# Contributing to pandas

First off, thank you for considering contributing to pandas! It's people like you that make pandas such a great tool.

## How to contribute

Pandas is a large project with a well-defined contribution process. To get started, please refer to our official **[Contributor's Guide](https://pandas.pydata.org/docs/development/index.html)** in the documentation.

The guide is the best place to find the most up-to-date information on:

- **Reporting bugs**: How to submit a bug report using our issue tracker.
- **Contributing code**: How to set up your development environment (including the recent migration to the Meson build system), follow our coding standards, and submit a pull request.
- **Contributing to documentation**: How to improve our docstrings and user guide.
- **Community**: How to get in touch with other contributors and users.

## Communication

- **GitHub Issues**: For bug reports and feature requests.
- **Community Slack**: For real-time discussion. Join via the [community page](https://pandas.pydata.org/docs/dev/development/community.html).
- **Mailing List**: For longer-form discussions and announcements.
- **Stack Overflow**: For usage questions (using the `pandas` tag).

## Code of Conduct

This project and everyone participating in it is governed by the [Pandas Code of Conduct](https://github.com/pandas-dev/pandas/blob/main/CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

Thank you for your interest in improving pandas!
